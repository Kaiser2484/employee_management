# Employee Management (OrangeHRM-style MVP)

Monorepo cho he thong quan ly nhan su (single-tenant), hien tai tap trung vao cac module:

- PIM (Employee Information Management)
- Leave Management
- Recruitment
- Profile/Settings
- Tasks
- Admin Catalogs

## 1. Tech Stack

- Backend: NestJS + TypeScript + Prisma
- Frontend: React + Vite + TypeScript
- Database: PostgreSQL
- Infra local: Docker Compose

## 2. Project Structure

```text
employee_management/
|- apps/
|  |- backend/          # NestJS API + Prisma
|  |- frontend/         # React app
|- packages/
|  |- shared/           # Shared package (reserved)
|- docs/
|  |- demo-accounts.txt
|- docker-compose.yml
|- .env.example
|- package.json
```

## 3. Prerequisites

- Node.js 20+
- npm 10+
- Docker Desktop

## 4. Environment Setup

Can tao cac file env sau:

1. Root env: copy `.env.example` thanh `.env`
2. Backend env: copy `apps/backend/.env.example` thanh `apps/backend/.env`
3. Frontend env: copy `apps/frontend/.env.example` thanh `apps/frontend/.env`

Gia tri quan trong can dong bo:

- `apps/backend/.env`:
	- `BACKEND_PORT=3002`
	- `DATABASE_URL=postgresql://postgres:postgres@localhost:5432/employee_management`
- `apps/frontend/.env`:
	- `VITE_API_BASE_URL=http://localhost:3002/api`

Note:

- Neu doi `BACKEND_PORT`, phai doi theo `VITE_API_BASE_URL`.
- Root `.env.example` dang de `3000`; du an hien tai dang default theo `3002` o backend/frontend.

## 5. Run Project Locally

### Step 1: Install dependencies

```bash
npm install
```

### Step 2: Start PostgreSQL

```bash
docker compose up -d
```

### Step 3: Apply migrations and seed data

```bash
npm run prisma:migrate --workspace @hrm/backend
npm run prisma:seed --workspace @hrm/backend
```

### Step 4: Run backend + frontend

```bash
npm run dev
```

Default URLs:

- Frontend: http://localhost:5173
- Backend API: http://localhost:3002/api
- Health: http://localhost:3002/api/health

## 7. API Overview (Current)

Base prefix: `/api`

Public:

- `GET /health`
- `POST /auth/login`
- `POST /auth/register`

Auth/Profile:

- `GET /auth/me`
- `GET /profile/me`
- `POST /profile/update`
- `POST /profile/password`
- `POST /profile/avatar`

Employees:

- `POST /employees`
- `GET /employees`
- `GET /employees/:employeeId`
- `PATCH /employees/:employeeId/role`

Leave:

- `GET /leave/requests`
- `POST /leave/requests`
- `PATCH /leave/requests/:requestId/approve`
- `PATCH /leave/requests/:requestId/reject`

Recruitment:

- `GET /recruitment/candidates`

Catalogs:

- `GET /catalogs`
- `POST /catalogs/:type`
- `DELETE /catalogs/:type`

Tasks:

- `POST /tasks`
- `GET /tasks`
- `GET /tasks/assignees`
- `GET /tasks/:taskId`
- `PATCH /tasks/:taskId`
- `PATCH /tasks/:taskId/status`
- `PATCH /tasks/:taskId/confirm-completion`

## 8. RBAC and Headers

He thong dang su dung role-scope thong qua headers:

- `x-user-id: <user-id>`
- `x-user-role: admin | hr | manager | team_lead | employee`

Scope behavior implemented:

- Team Lead: xem/duyet leave trong cung team
- Manager: xem/duyet leave trong cung department
- HR/Admin: truy cap toan bo
- Employee: chi truy cap du lieu ca nhan

## 11. Troubleshooting

1. Frontend goi API that bai do sai port
- Kiem tra `apps/backend/.env` va `apps/frontend/.env` co cung port (vd: `3002`).

2. Backend khong start do port dang bi chiem
- Script backend dev se thu kill process dang LISTEN tren `BACKEND_PORT` (Windows).

3. Prisma migrate loi ket noi DB
- Kiem tra `docker compose ps`
- Kiem tra `DATABASE_URL` trong `apps/backend/.env`

4. Login fail voi tai khoan demo
- Chay lai seed:

```bash
npm run prisma:seed --workspace @hrm/backend
```
